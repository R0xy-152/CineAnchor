# CineAnchor 产品需求文档 (PRD)

**版本**: v2.0  
**日期**: 2026-05-20  
**作者**: AI Product Manager  
**状态**: MVP 开发中

---

## 1. 产品一句话

> 一个网页工具，让任何人用自然语言生成 3D 场景，走进去自由探索，设计运镜，AI 渲染成电影级视频。

## 2. 解决的问题

### 用户痛点

| 用户 | 痛点 | 现有方案 | 缺陷 |
|------|------|----------|------|
| 3D 爱好者/创作者 | 想创作 3D 视频但学不会 Blender | Blender/Maya | 学习曲线 100h+ |
| AI 视频创作者 | Sora/Runway 生成的视频运镜不可控制 | Sora/Runway/Kling | 只能 prompt 描述运镜，无法精确操控 |
| 游戏概念美术师 | 需要快速把想法变成可探索的 3D 场景 | UE/Unity 手动搭场景 | 耗时 4-8h |
| 普通用户 | 想体验 AI 3D 但不知道从哪开始 | Meshy/Tripo 官网 | 只能生成模型，没有探索和视频输出 |

### CineAnchor 的解法

```
传统流程: 学 Blender (100h) → 建模 (4h) → 布光 (1h) → 调摄像机 (2h) → 渲染 (4h) → 视频
CineAnchor: prompt (10s) → AI 生成 3D (2min) → 自由探索 (2min) → 录制运镜 (2min) → AI 渲染 (5min) → 视频
```

**核心差异**: CineAnchor 不做最好的 AI 视频生成器，做最可控的 3D 导演工具。

## 3. 目标用户画像

### 第一优先级（MVP 面向）

**"有审美但不会 3D 工具的人"**

- 年龄: 18-35
- 场景: 想做酷炫的 3D 视频发社交媒体/作品集
- 技能: 会用手机拍视频，不会任何 3D 软件
- 需求: 快速出片、零学习成本、成果可分享

### 第二优先级（v2 拓展）

**"需要快速 Previz 的专业创作者"**

- 游戏/影视概念美术师
- 需要快速可视化场景构想
- 不需要最终渲染质量，需要速度和可控性

## 4. MVP 功能边界

### IN

- Text-to-3D（Meshy API，preview 模式）
- 网页 3D 取景器（Orbit/Drone/FPS 三模式）
- 关键帧录制 + Catmull-Rom 插值
- Camera path 保存/加载
- AI 视频渲染（ControlNet-Depth + AnimateDiff）
- 作品文件本地管理

### OUT (v2+)

- Image-to-3D
- 多 provider 3D API
- 用户账号系统
- 作品社区/Remix
- 物件布景/场景编辑器
- 移动端适配
- 云端 GPU 任务队列（先用本地 GPU）

## 5. 核心交互流程

```
1. 用户打开网页
2. 在首页看到产品介绍和使用流程
3. 点击"进入取景器"
4. 输入 prompt → 点击"AI 生成 3D 场景"
5. 等待 1-2min（Meshy API + 下载 GLB）
6. 3D 模型出现在取景器中，自动归一化大小
7. 用户用 WASD + 鼠标自由探索场景（FPS 模式）
8. 按 R 录制关键帧，在不同位置/角度捕获相机状态
9. Catmull-Rom 曲线自动插值，绿色轨迹实时显示
10. 保存运镜（命名 + 存到数据库）
11. 点击预览运镜 → 相机自动沿路径飞行
12. 输入视频 prompt → 提交渲染
13. GPU 渲染深度图 + ControlNet + AnimateDiff → MP4
14. 下载/查看最终视频
```

## 6. AI 技术选型

| 组件 | 选型 | 为什么 |
|------|------|--------|
| 3D 生成 | Meshy API (Text-to-3D) | 不自研模型，借力最成熟的 API。preview 模式 1-2min 出结果 |
| 深度图渲染 | gsplat (3DGS) | 提供精确的几何约束，比纯深度估计模型更准 |
| RGB 生成 | ControlNet-Depth (SD 1.5) | 深度图作为条件注入扩散模型，确保输出与场景结构对齐 |
| 时序一致 | AnimateDiff v2 | 跨帧注意力机制，消除闪烁抖动 |
| 帧平滑 | RAFT 光流插值 | 3x 帧倍增，画面更流畅 |

### 为什么用 3DGS 作为空间锚点

传统 AI 视频从噪声生成画面，没有三维空间参考。CineAnchor 用 3D 高斯泼溅从已有场景渲染精确深度图，ControlNet 将深度作为几何约束注入扩散模型。这意味着：

- 墙角是直的，不会扭曲
- 物体轮廓与 3D 模型一致
- 运镜过程中透视关系正确
- 多个镜头之间的空间连贯性有保证

## 7. 取舍决策

### 做了的

- 自研 3DGS+ControlNet 渲染管线（产品差异化核心）
- 完整的 3D 取景器（产品体验壁垒）
- Camera path 系统（可复用资产）

### 没做的

- **不自研 Text-to-3D 模型**: 成本高、效果不一定比 Meshy 好。我们做的是导演工具，不是 3D 基础模型
- **不做端到端视频生成**: Sora/Runway 已经在做且做得很好。我们的差异化在"空间可控性"
- **不做移动端**: Web 优先，Three.js/WebGL 移动端体验不好
- **不做实时协作**: MVP 阶段不需要。单用户体验先做透

## 8. 竞品分析

详见个人网站 Product Strategy 模块。

核心结论：**目前没有直接竞品**。最接近的是 AI 3D 生成 API (Meshy) + AI 视频生成 (Sora) + 3D 引擎 (Blender) 的组合，但没有人把它们整合成一个"零基础的 3D 导演工具"。

## 9. 成功指标（如果上线）

| 指标 | MVP 目标 | 说明 |
|------|----------|------|
| 从打开网页到生成第一个视频 | < 10min | 体验流畅度 |
| AI 3D 生成成功率 | > 90% | API 可靠性 |
| 视频渲染成功率 | > 95% | 管线稳定性 |
| 用户首次体验留存 | > 50% 会再回来 | 产品有持续价值 |
| 免费额度内可完成全流程 | 是 | 免费策略 |

## 10. 路线图

- **Phase 1 (当前)**: 核心闭环 — prompt → 3D → 取景器 → 运镜 → 视频
- **Phase 2**: 视频生成质量提升、GPU 管线稳定
- **Phase 3**: 作品展示页、Demo 视频制作
- **v2**: 用户系统、作品社区、Remix
- **v3**: 场景布景编辑、多 provider 切换
